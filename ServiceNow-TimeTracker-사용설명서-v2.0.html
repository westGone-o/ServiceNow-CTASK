📊 ServiceNow Time Tracker

버전 2.0.0 | 완전 사용 설명서

🎉 주요 업데이트

🎯 프로그램 소개

ServiceNow Time Tracker는 ServiceNow의 Service Request와 Change Request에 대한 Time Worked를 효율적으로 관리할 수 있는 데스크톱 애플리케이션입니다.

🗓️ 달력 기반 관리

직관적인 달력 인터페이스로 월별 작업 시간을 한눈에 확인

🎨 색상 구분

SR, CR, Closed 항목을 색상으로 구분하여 빠른 식별

✨ 드래그 앤 드롭

Task를 달력에 드래그하여 간편하게 Time Worked 생성

📊 실시간 진행률

예상 시간 대비 실제 소요 시간을 색상으로 표시

💾 로그인 기억

로그인 정보를 안전하게 저장하여 자동 로그인

🔗 빠른 이동

더블클릭으로 ServiceNow 페이지 바로 열기

🆕 버전 2.0 새로운 기능

✨ 주요 업데이트: 이번 버전에서는 사용자 편의성과 작업 효율성을 크게 향상시키는 기능들이 추가되었습니다!

1. 🔐 Remember Login (로그인 정보 저장)

  • 로그인 화면에서 "Remember Login" 체크박스 선택
  • 이메일과 비밀번호를 안전하게 암호화하여 로컬에 저장
  • 다음 실행 시 자동으로 로그인 정보가 입력됨
  • 보안: Base64 인코딩으로 안전하게 저장

2. 📧 이메일 기반 로그인

  • 기존 Username 대신 Email 주소로 로그인
  • 예: yourname@company.com
  • 더욱 직관적이고 편리한 로그인 방식

3. 📊 Change Task 진행률 표시

빨강: 예상 시간 초과 (실제 시간이 예상보다 많음)
  • 각 Change Task에 Estimated vs Actual 시간 표시
  • 예: CTASK0012345 - Task Name (예상 5h / 실제 7h) → 빨간색 표시
  • 예상 시간을 초과한 작업을 한눈에 파악 가능
  • 초과되지 않은 작업은 기본 파란색으로 표시

4. 🎨 Closed/Cancelled 항목 필터링

  • Active 항목만 목록에 표시: 완료된 CR/SR은 자동으로 숨김
  • 달력에서 회색으로 표시: Closed/Cancelled 항목은 회색으로 구분
  • 클릭 시 접근 가능: 회색 항목 클릭 시 해당 CR/SR 정보 자동 표시
Active Service Request (보라색)
Active Change Request (파란색)
Closed/Cancelled (회색)

5. 🔗 ServiceNow 페이지 바로가기

  • CR 또는 SR을 더블클릭하면 브라우저에서 해당 페이지 자동 열림
  • 빠르게 상세 정보를 확인하거나 업데이트 가능
  • 프로그램을 벗어나지 않고도 ServiceNow 연동

🚀 설치 및 실행

1단계: 압축 해제
  • ServiceNow-TimeTracker-v2.0.0.zip 파일을 원하는 위치에 압축 해제
  • 권장 위치: C:\Program Files\ServiceNow-TimeTracker
2단계: 실행
  • 압축 해제한 폴더에서 ServiceNow-TimeTracker.exe 더블클릭
  • 바탕화면에 바로가기 생성을 권장합니다
3단계: 첫 로그인
  • 로그인 창에 ServiceNow 계정 정보 입력
  • Base URL: https://one.service-now.com (회사 URL로 변경 가능)
  • Email: ServiceNow 이메일 주소 (예: yourname@company.com)
  • Password: ServiceNow 비밀번호
  • Remember Login: 로그인 정보를 저장하려면 체크
💡 팁: "Remember Login"을 체크하면 다음부터 자동으로 로그인됩니다!

🖥️ 화면 구성

📋 왼쪽 패널 - Service Request (SR)

  • 목록: 할당된 Active SR 목록 표시
  • 필터: 검색 기능으로 빠른 찾기
  • 색상: 보라색 배경
  • 정렬: SR 번호 순으로 자동 정렬
  • 더블클릭: ServiceNow SR 페이지로 이동
📌 참고: Closed/Cancelled 된 SR은 목록에 표시되지 않지만, 달력에서 클릭하면 자동으로 목록에 추가됩니다.

📋 가운데 패널 - Change Request (CR) & Change Task (CTASK)

  • 상단: Change Request 목록 (Active만)
  • 하단: 선택한 CR의 Change Task 목록
  • 색상: 파란색 배경
  • 진행률 표시: CTASK에 예상/실제 시간 및 색상 코드 표시
  • 더블클릭: ServiceNow CR 페이지로 이동

📊 CTASK 진행률 색상 코드

색상 의미 예시
🔴 빨강 예상 시간 초과 예상 5h, 실제 7h → 2h 초과로 빨간색 표시
🔵 파랑 정상 범위 예상 5h, 실제 3h 또는 5h → 기본 파란색 표시

📅 오른쪽 패널 - 달력

  • 월별 보기: 이전/다음 달 이동 버튼 (◀ ▶)
  • Time Worked 표시: 각 날짜별 작업 시간
  • 색상 구분:
    Active SR (보라색)
    Active CR (파란색)
    Closed/Cancelled (회색)
  • Total: 일별 총 작업 시간 자동 계산
  • 단일 클릭: 해당 Time Worked 및 CR/SR 자동 선택
  • 더블 클릭: Time Worked 수정 다이얼로그 열기
💡 팁: 달력의 회색 항목은 이미 완료된 CR/SR입니다. 클릭하면 과거 작업 내역을 확인할 수 있습니다!

📝 하단 패널 - Time Worked 목록

  • 왼쪽: 선택한 CR/CTASK의 Time Worked 목록
  • 오른쪽: 선택한 SR의 Time Worked 목록
  • 표시 내용: 날짜, 시간, 그룹, 코멘트
  • 더블클릭: 수정 다이얼로그

⚡ 주요 기능 상세 설명

1️⃣ Time Worked 생성 (드래그 앤 드롭)

  1. 항목 선택: 왼쪽 패널에서 SR 또는 CTASK 선택
  2. 드래그: 마우스로 항목을 클릭한 채로 드래그
  3. 드롭: 달력의 원하는 날짜에 드롭
  4. 정보 입력: Time Worked 생성 다이얼로그에서:
    • Date: 작업 날짜 (자동 설정, 수정 가능)
    • Time Spent: 작업 시간 (형식: 1h, 30m, 1h 30m)
    • Group: 작업 그룹 선택 (마지막 사용 그룹 자동 선택)
    • Comments: 작업 내용 (선택사항)
  5. 저장: Create 버튼 클릭
💡 팁: 마지막에 사용한 그룹이 자동으로 선택되어 연속 작업 시 시간을 절약할 수 있습니다!
📌 Time Spent 입력 형식:
  • 1h - 1시간
  • 30m - 30분
  • 1h 30m - 1시간 30분
  • 2h 45m - 2시간 45분

2️⃣ Time Worked 조회

방법 1: 목록에서 선택
  • SR 또는 CTASK를 선택
  • 하단에 해당 항목의 Time Worked 목록 자동 표시
방법 2: 달력에서 클릭
  • 달력의 Time Worked를 클릭
  • 해당 CR/SR/CTASK가 자동으로 선택됨
  • Time Worked 목록에서 해당 항목 하이라이트
  • 회색 항목 클릭 시: Closed CR/SR이 목록에 자동 추가됨
✨ 새로운 기능: 달력에서 Time Worked를 클릭하면 해당하는 CR/SR이 자동으로 왼쪽 목록에서 선택되고, 하단에 Time Worked 목록이 표시됩니다!

3️⃣ Time Worked 수정

방법 1: 목록에서 수정
  • 하단 Time Worked 목록에서 항목 더블클릭
  • 수정 다이얼로그가 열림
  • 정보 수정 후 Save 버튼 클릭
방법 2: 달력에서 수정
  • 달력의 Time Worked를 더블클릭
  • 수정 다이얼로그가 바로 열림
  • 가장 빠른 방법!
💡 팁: 달력의 Time Worked를 더블클릭하는 것이 가장 빠르고 편리합니다!
CR 또는 SR을 더블클릭하면:
  • 기본 브라우저에서 해당 ServiceNow 페이지가 자동으로 열림
  • 상세 정보를 확인하거나 직접 수정 가능
  • 프로그램과 ServiceNow를 오가며 작업 가능

4️⃣ ServiceNow 페이지 바로가기

CR 또는 SR을 더블클릭하면:
  • 기본 브라우저(Chrome)에서 해당 ServiceNow 페이지가 자동으로 열림
  • Chrome에 ServiceNow 계정이 로그인되어 있으면 별도 로그인 없이 바로 페이지 확인 가능
  • 상세 정보를 확인하거나 직접 수정 가능
  • 프로그램과 ServiceNow를 오가며 작업 가능
💡 팁: 평소 Chrome에서 ServiceNow를 사용하고 있다면, 더블클릭으로 즉시 로그인된 상태로 페이지를 확인할 수 있습니다!

5️⃣ Closed/Cancelled 항목 관리

자동 필터링:
  • Closed 또는 Cancelled 상태의 CR/SR은 목록에 표시되지 않음
  • 화면이 깔끔하게 유지되어 Active 작업에 집중 가능
달력에서 확인:
  • 완료된 작업의 Time Worked는 회색으로 표시
  • 과거 작업 내역을 시각적으로 확인 가능
접근 방법:
  • 달력의 회색 Time Worked 클릭
  • 해당 Closed CR/SR이 자동으로 목록에 추가됨
  • Time Worked 내역 확인 가능
💡 팁: 완료된 프로젝트의 시간을 확인하고 싶을 때, 달력에서 회색 항목을 클릭하면 바로 확인할 수 있습니다!

6️⃣ CTASK 진행률 확인

CTASK 목록에서 확인:
  • 각 CTASK 옆에 (예상시간 / 실제시간) 표시
  • 예: CTASK0012345 - Bug Fix (5h / 7h)
색상으로 상태 파악:
  • 빨강: 예상 시간을 초과함 (실제 > 예상) - 검토 필요
  • 파랑: 정상 범위 (실제 ≤ 예상) - 계획대로 진행 중
✨ 활용 방법: 빨간색 CTASK는 예상보다 시간이 오래 걸린 작업이므로, 원인 분석 및 향후 계획 수립 시 참고하세요!

시나리오 1: 일일 작업 기록
  1. 오늘 작업한 SR/CTASK를 선택
  2. 달력의 오늘 날짜에 드래그 앤 드롭
  3. 시간과 코멘트 입력 후 저장
  4. 총 작업 시간이 자동으로 계산되어 표시됨
시나리오 2: 월간 작업 내역 확인
  1. 달력에서 이전 달로 이동
  2. 각 날짜의 Total 시간 확인
  3. 회색 항목 클릭하여 완료된 프로젝트 내역 확인

  1. CR 선택하여 CTASK 목록 확인
  2. 빨간색 CTASK 확인 (예상 시간 초과된 작업)
  3. 예상 시간이 부족했던 원인 분석
  4. 향후 유사 작업 시 예상 시간 조정 참고

🔧 문제 해결

❌ 로그인 실패

증상: "Login failed" 메시지가 표시됨
  • ✅ ServiceNow 이메일 주소가 정확한지 확인
  • ✅ 비밀번호가 정확한지 확인 (대소문자 구분)
  • ✅ Base URL이 올바른지 확인 (https:// 포함)
  • ✅ 네트워크 연결 상태 확인
  • ✅ ServiceNow 웹사이트에 직접 로그인 시도
  • ✅ 회사 VPN 연결 확인 (필요한 경우)

❌ Remember Login이 작동하지 않음

  • 로그인 시 "Remember Login" 체크박스가 체크되었는지 확인
  • 프로그램 설치 폴더에 쓰기 권한이 있는지 확인
  • 프로그램을 완전히 종료한 후 다시 시작

❌ 데이터가 표시되지 않음

  • 상단 Refresh 버튼 클릭 (F5)
  • ServiceNow에 할당된 Active Task가 있는지 확인
  • 필터 설정이 너무 좁은지 확인
  • 로그를 확인하여 API 호출 오류가 있는지 확인

❌ 드래그 앤 드롭이 작동하지 않음

  • 항목을 정확하게 선택했는지 확인
  • 달력의 날짜 영역(흰색 부분)에 드롭했는지 확인
  • 프로그램 재시작
  • 마우스 드라이버 업데이트

❌ Time Worked 저장 실패

  • Time Spent 형식 확인: 1h, 30m, 1h 30m
  • 필수 항목이 모두 입력되었는지 확인 (Date, Time Spent, Group)
  • ServiceNow 연결 상태 확인
  • ServiceNow에서 Time Worked 생성 권한 확인

❌ 색상이 제대로 표시되지 않음

  • 데이터 새로고침 (F5)
  • CTASK의 Estimated Hours가 설정되어 있는지 확인
  • 프로그램 재시작

❌ ServiceNow 페이지가 열리지 않음

  • 기본 브라우저가 설정되어 있는지 확인
  • 브라우저가 정상 작동하는지 확인
  • ServiceNow URL 접근 권한 확인

❌ Closed CR/SR이 목록에 추가되지 않음

  • 달력의 회색 Time Worked를 클릭했는지 확인
  • 네트워크 연결 상태 확인
  • 로그를 확인하여 API 호출 오류 확인
💡 추가 도움이 필요한 경우: 문제가 지속되면 개발자에게 연락하여 로그 파일과 함께 상세한 오류 상황을 공유해주세요.

❓ 자주 묻는 질문 (FAQ)

Q1. Remember Login을 해제하려면 어떻게 하나요?

A: 로그인 화면에서 "Remember Login" 체크박스를 해제하고 로그인하면 저장된 정보가 삭제됩니다.

Q2. Time Spent는 어떤 형식으로 입력해야 하나요?

A: 1h, 30m, 1h 30m 형식으로 입력하세요. 시간과 분 사이에 공백을 넣어주세요.

Q3. Closed된 CR/SR은 어떻게 확인하나요?

A: 달력에서 회색으로 표시된 Time Worked를 클릭하면 해당 CR/SR이 목록에 자동으로 추가됩니다.

Q4. CTASK의 색상이 빨간색인데 어떻게 해야 하나요?

A: 빨간색은 예상 시간을 초과했음을 의미합니다. 이는 작업이 예상보다 오래 걸렸다는 표시이므로, 원인을 분석하여 향후 유사 작업 시 참고하세요.

Q5. ServiceNow 페이지가 열리지 않아요.

A: 기본 브라우저가 설정되어 있는지 확인하세요. Windows 설정에서 기본 브라우저를 지정할 수 있습니다.

Q6. 여러 달의 Time Worked를 한번에 확인할 수 있나요?

A: 달력의 이전/다음 버튼(◀ ▶)을 사용하여 월별로 이동하며 확인할 수 있습니다.

Q7. Time Worked를 잘못 입력했어요. 수정할 수 있나요?

A: 네! 달력이나 목록에서 해당 Time Worked를 더블클릭하면 수정 다이얼로그가 열립니다.

Q8. 프로그램이 느려지는 이유는 무엇인가요?

A: 많은 양의 Time Worked 데이터를 로드할 때 느려질 수 있습니다. 새로고침(F5)을 하면 최신 데이터로 갱신되면서 성능이 개선됩니다.

Q9. 다른 사용자에게 프로그램을 공유해도 되나요?

A: 네, 압축 파일을 공유하면 됩니다. 각 사용자는 자신의 ServiceNow 계정으로 로그인하여 사용합니다.

Q10. 프로그램 업데이트는 어떻게 하나요?

A: 새 버전이 출시되면 개발자로부터 공지를 받게 됩니다. 새 버전의 압축 파일을 다운로드하여 기존 폴더에 덮어쓰면 됩니다.
